# M51-1/2 MARTIN JÉRÉMY - UX pour les objets connectés et l’affichage d’informations sur l’énergie

# ETAT INITIAL ET OBJECTIFS À RENDRE POUR LE 4 MARS 23h59

## Mon sujet d'étude est ...

Mon sujet d’étude porte sur les meilleures pratiques UX pour les objets connectés, avec un focus particulier sur l'affichage des données énergétiques. L’objectif est d’identifier les principes UX essentiels pour rendre ces interfaces compréhensibles, engageantes et actionnables par l’utilisateur final.

### Contexte

Ce sujet est directement lié à mon Travail de Bachelor, qui vise à concevoir une interface tangible permettant la visualisation des consommations énergétiques.
Une UX bien pensée est essentielle pour garantir que :

- Les utilisateurs comprennent facilement leurs consommations énergétiques.
- L’affichage des informations soit intuitif et utile.
- Les utilisateurs puissent adopter des comportements plus éco-responsables grâce à une UX bien conçue.

Ce sujet me permettra également de renforcer mes compétences en UX design adaptées aux objets connectés, ce qui est une expertise recherchée dans le domaine de la smart home et des interfaces immersives.

## État initial

Actuellement, mes connaissances en UX concernent principalement le web et les applications mobiles. J’ai une bonne maîtrise des méthodes UX standards (personas, wireframes, tests utilisateurs, heuristiques UX, design thinking), mais je manque d'expérience sur les spécificités UX des objets connectés, notamment :

- L'affichage des données énergétiques : comment les rendre compréhensibles et exploitables ?
- Le feedback utilisateur dans les interfaces tangibles : comment utiliser des signaux visuels, sonores ou haptiques pour améliorer l'expérience ?
- Les biais cognitifs et comportements liés à la consommation énergétique.

L’objectif de cette auto-formation est donc d’approfondir ces aspects et de voir comment ces principes peuvent être appliqués à mon projet de Bachelor.

## Objectifs

- Synthétiser 3 à 5 principes clés de l’UX pour les objets connectés et l’affichage des données énergétiques d’ici le 10 mars.
- Analyser 2 exemples d’objets connectés liés à l’affichage énergétique et identifier leurs forces/faiblesses UX d’ici le 15 mars.
- Créer une maquette statique (Figma, Miro, Notion, etc.) illustrant une bonne pratique UX pour l’affichage énergétique d’ici le 20 mars.
- Rédiger une courte synthèse des enseignements tirés et des recommandations pour mon Travail de Bachelor d’ici le 25 mars.

## Démarche

- Phase 1 (6h - jusqu’au 10 mars) : Recherche des bonnes pratiques UX pour les objets connectés et l’affichage énergétique (lectures, études UX, benchmarks).
- Phase 2 (6h - jusqu’au 15 mars) : Analyse de 2 objets connectés existants (ex : Google Nest, Glow, Home Assistant).
- Phase 3 (8h - jusqu’au 20 mars) : Création d’une maquette statique illustrant une bonne pratique UX pour l’affichage énergétique.
- Phase 4 (6h - jusqu’au 25 mars) : Rédaction d’une synthèse sur les enseignements tirés et les recommandations pour mon TB.
- Phase 5 (4h - jusqu’au 28 mars) : Mise en forme du rapport final et préparation de la restitution.

## 5 questions

1. Quels sont les défis UX spécifiques aux objets connectés par rapport aux interfaces classiques (web/mobile) ?
2. Quelles sont les caractéristiques essentielles d’une bonne datavisualisation pour la consommation énergétique ?
3. Comment utiliser les feedbacks sensoriels (visuel, sonore, haptique) pour améliorer l’expérience des objets connectés ?
4. Quels biais cognitifs influencent la perception des données énergétiques par les utilisateurs ?
5. Comment structurer un test utilisateur UX pour évaluer une interface d’objet connecté sans prototype fonctionnel ?

## Expérimentation

Au lieu de concevoir un prototype réel, je vais réaliser une maquette statique illustrant une ou plusieurs bonnes pratiques UX pour l'affichage des données énergétiques.

Format possible :
- Wireframe détaillé sur Figma ou Miro montrant comment afficher clairement et efficacement des données de consommation.
- Mockup d’une interface optimisée en fonction des principes UX étudiés.
- Modèle interactif simple montrant différentes options d’affichage et d’interaction utilisateur (sans implémentation technique).

L’objectif est d’appliquer concrètement les bonnes pratiques UX identifiées dans la recherche, sans nécessiter de développement ou de prototype physique.
