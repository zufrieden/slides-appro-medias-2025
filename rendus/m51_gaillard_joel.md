## Démarche

_Quelles étapes et comment allez-vous répartir l'investissement des 36h disponibles pour le cours ?_

**Étape 1 (environ 15h)**
Je vais commencer par une étude théorique : consulter diverses ressources (articles, vidéos, forums, tutoriels) sur les patterns afin de clarifier leur définition, leurs avantages et leurs limites. Je prendrai aussi le temps de manipuler des exemples de code pour bien comprendre chaque pattern.

**Étape 2 (environ 15h)**
Je poursuivrai avec l’analyse pratique dans des frameworks open-source afin de repérer concrètement la présence de ces patterns. J’illustrerai mes observations par de courts extraits de code et noterai comment ceci est mis en œuvre. Je ferai un rapprochement avec Rumya en tentant de deviner comment les patterns sont utilisés pour me préparer à mieux comprendre lors de mon travail de bachelor.

**Étape 3 (environ 6h)**
Je rédigerai un rapport rassemblant l’ensemble de mes découvertes. Il présentera les design patterns majeurs, des exemples concrets tirés de Laravel et les hypothèses que j’aurai dégagées pour Rumya. Ce document me servira également de guide méthodologique, expliquant comment reconnaître ces patterns dans tout nouveau framework.
